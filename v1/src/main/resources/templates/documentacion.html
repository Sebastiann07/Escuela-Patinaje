Descripción del Proyecto

Sistema web completo para la gestión administrativa de una escuela de patinaje.

Desarrollado con Spring Boot, este sistema permite gestionar usuarios, instructores, clases y horarios de forma eficiente y segura.

Características Principales:
Gestión de Usuarios

Sistema completo de registro, autenticación y roles (ADMIN, INSTRUCTOR, ALUMNO)

Gestión de Clases

Administración de horarios, niveles, capacidad e inscripciones

Gestión de Instructores

Control de especialidades, experiencia y asignación a clases

APIs RESTful

Endpoints JSON para integración con aplicaciones externas

Guía de Instalación

Requisitos Previos
  • Java Development Kit (JDK) 17 o superior
  • MySQL 8.0 o superior
  • Maven 3.6 o superior
  • IDE recomendado: IntelliJ IDEA, Eclipse o VS Code
Pasos de Instalación
1. Clonar el Repositorio
git clone https://github.com/tu-usuario/skatepro-academy.git
cd skatepro-academy
2. Configurar Base de Datos

Crear la base de datos en MySQL:

CREATE DATABASE escuela_patinaje;

CREATE USER 'patinaje_user'@'localhost' IDENTIFIED BY '1234';
GRANT ALL PRIVILEGES ON escuela_patinaje.* TO 'patinaje_user'@'localhost';
FLUSH PRIVILEGES;
3. Configurar application.properties

Editar src/main/resources/application.properties:

server.port=8080

spring.datasource.url=jdbc:mysql://localhost:3306/escuela_patinaje
spring.datasource.username=patinaje_user
spring.datasource.password=1234

spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true
spring.jpa.properties.hibernate.dialect=org.hibernate.dialect.MySQLDialect
4. Compilar y Ejecutar
# Compilar el proyecto
mvn clean install

# Ejecutar la aplicación
mvn spring-boot:run
5. Acceder a la Aplicación

Abrir el navegador en: http://localhost:8080

Configuración y Seguridad

Sistema de Autenticación

El sistema utiliza Spring Security con las siguientes características:

  • Autenticación basada en formulario (Form-based Login)
  • Contraseñas encriptadas con BCrypt
  • Gestión de sesiones HTTP
  • Protección CSRF (excepto para endpoints API)
Roles y Permisos
Rol Permisos
ADMIN Acceso total: gestión de usuarios, clases, instructores
INSTRUCTOR Gestión de sus clases, visualización de alumnos
ALUMNO Visualización de clases, inscripciones, perfil personal
Rutas Protegidas
// Rutas públicas
/, /index/**, /registro, /login, /css/**, /js/**, /images/**

// Solo ADMIN
/users/**

// INSTRUCTOR y ADMIN
/instructor/**

// ALUMNO, INSTRUCTOR y ADMIN
/alumno/**

Modelos de Datos

UserModel (tbl_users)
Campo Tipo Descripción
idLongIdentificador único (PK, Auto)
nameString(50)Nombre completo del usuario
emailString(100)Correo electrónico (Único)
passwordStringContraseña encriptada (BCrypt)
phoneString(15)Número de teléfono
fechaNacimientoLocalDateFecha de nacimiento
generoString(20)Género del usuario
rolEnumADMIN, INSTRUCTOR, ALUMNO
activoBooleanEstado de la cuenta (true/false)
InstructorModel (tbl_instructores)
Campo Tipo Descripción
idLongIdentificador único (PK, Auto)
nombreString(50)Nombre del instructor
emailString(50)Correo electrónico (Único)
telefonoString(15)Número de contacto
especialidadString(100)Ej: Freestyle, Velocidad, Artístico
experienciaString(100)Años de experiencia
estadoString(20)Activo/Inactivo

ClaseModel (tbl_clases)
Campo Tipo Descripción

diaString(20)Lunes, Martes, Miércoles, etc.

horaFinString(20)Hora de finalización (HH:mm)

inscritosIntegerNúmero actual de inscritos
descripcionTextDescripción de la clase
estadoString(20)Activa, Inactiva, Pausada
instructorManyToOneRelación con instructor (FK)

Seguridad y Mejores Prácticas

Características de Seguridad Implementadas
  • Encriptación de contraseñas con BCrypt
  • Protección CSRF en formularios
  • Control de acceso basado en roles (RBAC)
  • Validación de datos con Bean Validation
  • Sesiones HTTP seguras
  • Logout con invalidación de sesión
  • Protección contra ataques XSS
  • Prevención de SQL Injection (JPA)
Recomendaciones de Seguridad
Importante: En producción, asegúrate de:
  • Cambiar las credenciales de la base de datos
  • Usar HTTPS para las comunicaciones
  • Configurar variables de entorno para datos sensibles
  • Implementar rate limiting en los endpoints
  • Mantener las dependencias actualizadas
